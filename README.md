# list_with_random_links
ТЗ выполнил следующим образом:
1. Для реализации списка создал три класса (List, ListNode, ListNodeData).
  - класс ListNodeData - обертка над строкой, введена для проверки и контроля ограничений (длина и формат UTF8) на строку данных.
  - класс ListNode - непосредственно требуемый в ТЗ класс, как и требовалось в задании состоит из четверки (никаких дополнительных полей не добавлял) указанных полей, однако для инкапсуляции и реализации ограничений (для избежания некорректных операций над разными экземплярами списка) весь функционал чтения и модификации приватный. 
  - класс List - собственно сам класс представляющий минимальный интерфейс двусвязного списка с произвольными связями, в этом классе полностью сосредоточена вся логика получения данных из списка и его модификации.  
2. Структура реализации состоит из 2 папок: в src содержатся реализация и заголовочные файлы классов, в tests содержатся первичные тесты соответствующих реализаций классов.

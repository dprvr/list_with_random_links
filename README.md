# ListWithRandomLinks
### Постановка задачи
Требуется реализовать *двунаправленный список с дополнительной произвольной ссылкой*. Каждый узел такого списка состоит из четырех полей:
```c++
struct ListNode { 
	ListNode* prev;
	ListNode* next;
	ListNode* rand;
	std::string data;
};
```
где *prev *- указатель на предыдущий узел списка, *next* - указатель на следующий узел списка, *rand* - указатель на произвольный узел *этого же списка*, *data* - произвольные данные в виде строки (в кодировке UTF-8). Изменять множество полей узла *ListNode* запрещено.

Интерфейс списка должен предоставлять следующие возможности:
1. Добавлять новые узлы в произвольную позицию списка.
2. Удалять существующие узлы.
3. Осуществлять перемещение между узлами в обоих направлениях.
4. Для узлов устанавливать произвольные ссылки и данные.
5. Получать текущее количество узлов в списке.
6. Создавать список из текстового файла, состоящего из строк формата "<строка данных>;<номер узла на который указывает произвольная ссылка либо -1 при ее отсутствии>".
7. Проводить сериализацию и десериализацию списка в бинарный файл.

При этом максимальное количество узлов может составлять 10^6 узлов, длина каждой строки данных data может быть до 1000 символов.
### Описание решения
Реализация выполнена на языке c++ 17. Для проверки корректности формата UTF-8 использована библиотека UTF8-CPP.  Сборка осуществляется с помощью cmake. Структура реализации состоит из 2 папок:
1. src - содержит заголовочные файлы (в папке heads) и файлы реализации (в папке impl).
2. tests - содержит первичные тесты реализации.

Двунаправленный список реализован в виде 3-х классов:
1. ListNodeData - класс реализующий паттерн ValueObject для  данных узла, введен для осуществления контроля ограничений на данные узла.
2. ListNode - класс реализующий непосредственно узел списка, содержит только требуемые в задаче поля. При этом для инкапсуляции данных класса и реализации ограничений узлов (операции над узлами должны производиться в рамках одного экземпляра списка) весь функционал данного класса приватный (доступен только дружественному классу List).
3. List - класс реализующий собственно сам список, в нем сосредоточена вся логика работы с самим списком и его узлами.
